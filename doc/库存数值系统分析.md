## 一、字段全景表（真实含义 vs 字面命名）

| 字段名 | 字面意思 | 真实含义 | 数据来源 | 是否实时更新 |
|--------|----------|----------|----------|-------------|
| `total_stock` | 总库存 | **汇总快照值**，不是实时"总库存"，而是多个流程字段的快照合计，有时可能与实际不符 | 程序手动赋值 | ❌（需触发重计算） |
| `onhand_stock` | 在手库存 | **实物已上架在库量**，已上架到标准货位，可直接出货的量 | GoodsLocation（标准位）汇总 | ❌（putaway/removal 后同步） |
| `locked_stock` | 锁定库存 | **onhand 中被 DN 单软预留的量**，实物还在货架上，只是逻辑上不允许其他人卖 | DN 创建时 +，close/complete 时 - | ✅ 每次 DN 操作实时 +/- |
| `damage_stock` | 损坏库存 | **实物在残次品货位的量**，已上架但标记为残次 | GoodsLocation（damaged 位）汇总 | ❌（putaway/adjustment 后同步） |
| `return_stock` | 退货库存 | **实物在退货货位的量**，退货待复核或处理 | GoodsLocation（return 位）汇总 | ❌（putaway/adjustment 后同步） |
| `asn_stock` | ASN 库存 | **还未到仓的在途数量**，pending 状态 ASN 明细的合计，是对"将来会收到多少"的预告 | 重新查 ASN 表计算 | ❌（创建/关闭 ASN 后重算） |
| `received_stock` | 已收货库存 | **卸货已签收、但还没分拣的数量**，是个短暂的中间状态，分拣完成后归零 | ASN 签收 +，分拣完成 - | ✅ |
| `sorted_stock` | 已分拣库存 | **分拣完成但还没上架的数量**，等待上架工人将其放入货位 | 分拣 +，上架 -，下架异常 + | ✅ |
| `dn_stock` | DN 库存 | **还未拣货的出库预扣量**，pending 状态 DN 的合计，代表"已承诺要出但还没动手" | 重新查 DN 表计算 | ❌（创建/关闭 DN 后重算） |
| `picked_stock` | 已拣货库存 | **已从货架拣出但还没打包的数量** | 拣货完成 +，打包完成 - | ✅ |
| `packed_stock` | 已打包库存 | **已打包但还未发出的数量**，在打包区等待发货 | 打包完成 +，发货完成 - | ✅ |
| `delivered_stock` | 已发货库存 | **已交给承运商但客户尚未签收的数量**，在途中 | 发货 +，客户签收 - | ✅ |
| `low_stock_threshold` | 低库存阈值 | **补货触发线**，`-1` 表示禁用 | 人工配置 | — |
| `high_stock_threshold` | 高库存阈值 | **积压告警线**，`-1` 表示禁用 | 人工配置 | — |

## 二、两个计算属性的真实含义

```
available_stock_for_sale = onhand_stock - locked_stock - dn_stock

available_stock          = onhand_stock + damage_stock + return_stock
                                        - locked_stock - dn_stock
```

| 属性 | 直白含义 | ⚠️ 潜在问题 |
|------|----------|------------|
| `available_stock_for_sale` | 纯净可卖出的量（在库、未被任何单占用） | 命名最准确 |
| `available_stock` | 把残次品和退货都算进"可用" | **有意设计**：本系统中损坏品和退货品均可正常出库，因此纳入可用量计算 |

## 三、total_stock 的真实计算公式

```
total_stock = onhand_stock
            + damage_stock
            + return_stock
            + sorted_stock      ← 还在过渡区，未上架
            + picked_stock      ← 已离架，在拣货区
            + packed_stock      ← 已打包，在打包区
```

> ⚠️ **`delivered_stock` 不在 total_stock 里！**
> 已交给承运商的货物不计入仓库总库存，物权已转移。
>
> ⚠️ **`asn_stock`、`received_stock`、`dn_stock` 也不在 total_stock 里！**
> - `asn_stock`：货还在供应商那里
> - `received_stock`：极短暂的中间状态（分拣前）
> - `dn_stock`：是对 onhand_stock 的软预留，不额外计数

## 四、ASN 入库流程——数值变化全链路

```
╔══════════════════════════════════════════════════════════════════════════════════╗
║  ASN 入库流程                                                                    ║
╠══════════════╦═══════════════════════════════════════════════════════════════════╣
║  阶段        ║  数值变化                                                         ║
╠══════════════╬═══════════════════════════════════════════════════════════════════╣
║ 创建 ASN     ║  asn_stock += quantity（重新从 ASN 表聚合计算）                    ║
╠══════════════╬═══════════════════════════════════════════════════════════════════╣
║ 签收 ASN     ║  asn_stock     -= quantity                                        ║
║ (receive)    ║  received_stock += quantity                                        ║
╠══════════════╬═══════════════════════════════════════════════════════════════════╣
║ 分拣完成     ║  received_stock -= quantity   (若 < 0 则强制置 0)                 ║
║ (complete)   ║  sorted_stock   += actual_quantity  (可能 ≠ quantity)             ║
║              ║  total_stock 重新计算                                              ║
╠══════════════╬═══════════════════════════════════════════════════════════════════╣
║ 上架完成     ║  sorted_stock -= quantity                                          ║
║ (putaway)    ║  GoodsLocation.quantity += quantity                               ║
║              ║  → 触发 update_and_calculate_stock：                               ║
║              ║    onhand_stock = GoodsLocation(standard) 之和                    ║
║              ║    damage_stock = GoodsLocation(damaged)  之和                    ║
║              ║    return_stock = GoodsLocation(return)   之和                    ║
║              ║    total_stock 重新计算                                             ║
╚══════════════╩═══════════════════════════════════════════════════════════════════╝
```

**数量流向（ASN）：**

```
供应商         asn_stock     received_stock    sorted_stock    GoodsLocation
  ○──────────►[       ]─────────────►[       ]──────────►[       ]──────────►[onhand]
  创建ASN时       签收时                  分拣时                 上架时
```

## 五、DN 出库流程——数值变化全链路

```
╔══════════════════════════════════════════════════════════════════════════════════╗
║  DN 出库流程                                                                     ║
╠══════════════╦═══════════════════════════════════════════════════════════════════╣
║  阶段        ║  数值变化                                                         ║
╠══════════════╬═══════════════════════════════════════════════════════════════════╣
║ 创建 DN      ║  dn_stock += quantity（重新从 DN 表聚合计算）                      ║
║              ║  lock_inventory：locked_stock += quantity                          ║
╠══════════════╬═══════════════════════════════════════════════════════════════════╣
║ 拣货完成     ║  dn_stock     -= quantity        (⚠️ 扣的是预扣量，见问题 #2)     ║
║ (picking)    ║  picked_stock += picked_quantity  (实际拣货量)                    ║
║              ║  GoodsLocation.quantity -= picked_quantity                         ║
║              ║  → 触发 update_and_calculate_stock（onhand 更新）                  ║
║              ║  total_stock 重新计算                                               ║
╠══════════════╬═══════════════════════════════════════════════════════════════════╣
║ 打包完成     ║  picked_stock -= packed_quantity                                   ║
║ (packing)    ║  packed_stock  += packed_quantity                                  ║
║              ║  total_stock 重新计算                                               ║
╠══════════════╬═══════════════════════════════════════════════════════════════════╣
║ 发货完成     ║  packed_stock    -= delivered_quantity                              ║
║ (delivery)   ║  delivered_stock += delivered_quantity                              ║
║              ║  total_stock 重新计算                                               ║
╠══════════════╬═══════════════════════════════════════════════════════════════════╣
║ 客户签收     ║  delivered_stock -= quantity                                        ║
║ (complete)   ║  total_stock 重新计算                                               ║
║              ║  unlock_inventory：locked_stock -= quantity                         ║
╠══════════════╬═══════════════════════════════════════════════════════════════════╣
║ 关闭 DN      ║  dn_stock 重新聚合计算（close 后 DN 从 pending 移出）               ║
║ (close)      ║  unlock_inventory：locked_stock -= quantity                         ║
╚══════════════╩═══════════════════════════════════════════════════════════════════╝
```

**数量流向（DN）：**

```
GoodsLocation    dn_stock(预扣)   picked_stock    packed_stock    delivered_stock   客户
[onhand]──────►[           ]────►[           ]──►[           ]──►[             ]──►  ✓
  创建DN                  拣货完成         打包完成         发货完成          签收完成
```

## 六、其他操作对数值的影响

| 操作 | 受影响字段 | 变化逻辑 |
|------|-----------|---------|
| **库存调整（Adjustment）** | `onhand_stock` / `damage_stock` / `return_stock` + `total_stock` | 先改 GoodsLocation，再触发 update_and_calculate_stock 同步 |
| **循环盘点（CycleCount）** | 记录 `system_quantity`（快照）+ `difference`（差异） | 不直接改 Inventory，只记录差异；需手工发起 Adjustment 才会真正改库存 |
| **下架（Removal）** | `sorted_stock` ↑，GoodsLocation.quantity ↓，随后触发 update_and_calculate_stock | onhand/damage/return 同步更新 |
| **库存快照（Snapshot）** | 无（只读） | 拍下 GoodsLocation.quantity 的副本，时间切片存档 |

## 八、库存页面各字段究竟显示什么

| 页面展示字段 | 对应字段/计算 | 业务解读 |
|-------------|-------------|---------|
| 总库存 | `total_stock` | 仓库内所有形态的实物合计（含过渡区货物） |
| 可用库存 | `available_stock` | 可以对外承诺的量（含残次品和退货，减去已预留） |
| 可售库存 | `available_stock_for_sale` | 真正可卖的净量（只算标准在库，减去预留） |
| 锁定库存 | `locked_stock` | 已被 DN 单占用但实物未动的量 |
| 在途入库 | `asn_stock` | 供应商已生成 ASN 但货物还未到仓的量 |
| 残次品 | `damage_stock` | 在损坏货位的实物量 |
| 退货库存 | `return_stock` | 在退货货位待处理的实物量 |

## 十、已识别的语义问题

| # | 问题描述 | 影响范围 |
|---|---------|---------|
| 1 | `available_stock` 包含 `damage_stock` 和 `return_stock`，**属于有意设计**：损坏品和退货品在本系统中仍可出库 | 无问题 |
| 2 | `dn_picked` 中用原始预扣量 `quantity` 扣减 `dn_stock`，而非实际拣货量 `picked_quantity`——缺货拣不满时 `dn_stock` 会被多扣 | dn_stock 计算偏差 |
| 3 | `total_stock` 不含 `delivered_stock`——已发货在途货物不算入仓库总量，此设计是否有意为之需确认（部分 WMS 会把"在途出货"也计入监控） | 语义层面待确认 |
| 4 | `asn_completed` 中 `received_stock` 若变负则强制置 0，说明系统允许分拣量超过签收量（多拣），但丢失了超拣的准确信息 | received_stock 数据不精确 |
| 5 | `locked_stock` 和 `dn_stock` 对同一批货物双重预扣：DN 创建时两者同步 +，`available_stock_for_sale = onhand - locked - dn` 实际对这部分货物扣了两次，可能导致可售量被过度压缩 | 可售量计算偏低 |
