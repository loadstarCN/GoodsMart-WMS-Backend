# 库存数值系统说明

> 适用模块：`warehouse/inventory/`
> 关联模型：`Inventory`、`GoodsLocation`

本文档说明库存页面所有数值字段的含义、计算方式，以及入库/出库各操作环节对这些数值的影响。阅读前需了解：系统中的库存数值分为两类——**实物位置类**（反映货物在哪个货位）和**流程过渡类**（反映货物正处于哪个业务环节）。

---

## 一、字段说明

### 1. 实物位置类（由 GoodsLocation 同步）

这三个字段的数据来源是 `GoodsLocation` 表，每次上架、下架、库存调整完成后会触发 `update_and_calculate_stock` 重新从货位汇总。

| 字段 | 含义 |
|------|------|
| `onhand_stock` | 已上架到**标准货位**的实物数量，是日常库存管理的核心字段 |
| `damage_stock` | 已上架到**残次品货位**的实物数量。本系统中残次品仍可正常出库 |
| `return_stock` | 已上架到**退货货位**的实物数量。本系统中退货品仍可正常出库 |

### 2. 入库流程过渡类（ASN）

这三个字段随 ASN 入库流程流转，货物完成上架后归零。

| 字段 | 含义 |
|------|------|
| `asn_stock` | 供应商已生成 ASN、**货物尚未到仓**的计划数量。从所有 `pending` 状态的 ASN 明细聚合计算，不代表实物 |
| `received_stock` | 包裹已到仓签收、**尚未分拣**的计划数量。注意：签收时按 ASN 计划量记账，实物数量以分拣为准 |
| `sorted_stock` | 分拣完成、**尚未上架**的实物数量。此阶段货物已清点，数量准确 |

### 3. 出库流程过渡类（DN）

这四个字段随 DN 出库流程流转，货物完成客户签收后归零。

| 字段 | 含义 |
|------|------|
| `dn_stock` | 已创建 DN 单、**尚未开始拣货**的预扣数量。从所有 `pending` 状态的 DN 明细聚合计算。货物实物未动，仅做逻辑占位 |
| `picked_stock` | 已从货架拣出、**尚未打包**的实物数量 |
| `packed_stock` | 已打包完成、**尚未发货**的实物数量 |
| `delivered_stock` | 已交给承运商、**客户尚未签收**的在途数量。货物一旦发出即不再计入仓库总库存（有意设计） |

### 4. 汇总与阈值

| 字段 | 含义 |
|------|------|
| `total_stock` | 仓库内所有形态实物的合计（见下方公式），由程序在每次流程节点推进后重新计算 |
| `locked_stock` | `onhand_stock` 中被**手动冻结**的数量，当前预留给将来的手动冻结功能，业务流程中暂不使用 |
| `low_stock_threshold` | 补货告警阈值，`-1` 表示禁用 |
| `high_stock_threshold` | 积压告警阈值，`-1` 表示禁用 |

## 二、计算公式

### total_stock（仓库总库存）

```
total_stock = onhand_stock    ← 标准货位
            + damage_stock    ← 残次品货位
            + return_stock    ← 退货货位
            + sorted_stock    ← 分拣区，等待上架
            + picked_stock    ← 拣货区，已离架
            + packed_stock    ← 打包区，等待发货
```

不含 `asn_stock`（货在供应商处）、`received_stock`（极短暂中间态）、`dn_stock`（软预留，不额外计数）、`delivered_stock`（已出仓）。

### available_stock（可出库量）

```
available_stock = onhand_stock + damage_stock + return_stock
                - locked_stock - dn_stock
```

包含残次品和退货品，因为本系统中这两类货物均可正常出库。减去 `locked_stock`（手动冻结）和 `dn_stock`（已被 DN 单占用）。

### available_stock_for_sale（纯净可售量）

```
available_stock_for_sale = onhand_stock - locked_stock - dn_stock
```

只计算标准货位的库存，适用于仅面向正常商品的销售场景。

> **注意**：`locked_stock` 当前始终为 0（手动冻结功能尚未上线），因此目前两个可用量的差值就是 `damage_stock + return_stock`。

## 三、库存页面字段对照

| 页面显示 | 对应字段 | 说明 |
|---------|---------|------|
| 总库存 | `total_stock` | 仓内所有形态实物合计 |
| 可用库存 | `available_stock` | 含残次/退货，减去预留 |
| 可售库存 | `available_stock_for_sale` | 仅标准在库，减去预留 |
| 在途入库 | `asn_stock` | 供应商已出单但未到仓 |
| 残次品 | `damage_stock` | 残次货位实物量 |
| 退货库存 | `return_stock` | 退货货位实物量 |
| 锁定库存 | `locked_stock` | 手动冻结量（当前为 0） |

## 四、ASN 入库流程——数值变化全链路

```
阶段            操作                        数值变化
─────────────────────────────────────────────────────────────────────
创建 ASN        pending ASN 明细写入        asn_stock  += 计划量
                                            （从 ASN 表重新聚合）

包裹签收         receive_asn                 asn_stock      -= 计划量
                                            received_stock += 计划量
                注意：此处按计划量记账，
                非实物清点数量

分拣完成         complete_asn                received_stock -= 计划量
（sorting task   （由分拣任务完成时           sorted_stock   += 实际分拣量
 complete 触发） 自动触发）                  total_stock 重新计算

                实际分拣量 ≠ 计划量时：
                  少于计划 → sorted_stock 小于预期，差额核销
                  多于计划 → sorted_stock 超出计划，超出量正常入库

上架完成         putaway_completed           sorted_stock -= 上架量
                                            GoodsLocation.quantity += 上架量
                                            触发 update_and_calculate_stock：
                                              onhand_stock = GoodsLocation(standard) 汇总
                                              damage_stock = GoodsLocation(damaged)  汇总
                                              return_stock = GoodsLocation(return)   汇总
                                            total_stock 重新计算
─────────────────────────────────────────────────────────────────────
```

**流向示意：**

```
供应商       asn_stock    received_stock   sorted_stock   GoodsLocation
  ○─────────►[     ]──────────────►[     ]─────────►[     ]──────────►[onhand]
          创建ASN       签收时              分拣完成          上架完成
                      （计划量）          （实际量）
```

## 五、DN 出库流程——数值变化全链路

```
阶段            操作                        数值变化
─────────────────────────────────────────────────────────────────────
创建 DN         pending DN 明细写入         dn_stock += 计划量
                                            （从 DN 表重新聚合）

进行中          progress_dn                 无库存变化，仅状态流转

拣货完成         picking_dn                  dn_stock      -= 实际拣货量
                                            picked_stock  += 实际拣货量
                                            GoodsLocation.quantity -= 实际拣货量
                                            触发 update_and_calculate_stock
                                            total_stock 重新计算

                缺货拣不满时：
                  dn_stock 按实际拣货量扣减，
                  剩余未拣量继续保留在 dn_stock 中

打包完成         packing_dn                  picked_stock  -= 打包量
                                            packed_stock  += 打包量
                                            total_stock 重新计算

发货完成         delivery_dn                 packed_stock    -= 发货量
                                            delivered_stock += 发货量
                                            total_stock 重新计算

客户签收         complete_dn                 delivered_stock -= 签收量
                                            total_stock 重新计算

关闭 DN          close_dn                    dn_stock 重新聚合计算
（pending 状态）                             （DN 移出 pending 后自动减少）
─────────────────────────────────────────────────────────────────────
```

**流向示意：**

```
GoodsLocation  dn_stock(预扣)  picked_stock  packed_stock  delivered_stock  客户
[onhand]──────►[          ]──►[          ]─►[          ]─►[            ]──► ✓
           创建DN        拣货完成       打包完成       发货完成         签收完成
```

## 六、其他操作对库存的影响

| 操作 | 影响字段 | 说明 |
|------|---------|------|
| **库存调整（Adjustment）** | `onhand_stock` / `damage_stock` / `return_stock` / `total_stock` | 直接修改 GoodsLocation，完成后触发 `update_and_calculate_stock` 同步到 Inventory |
| **循环盘点（CycleCount）** | 不直接改 Inventory | 盘点只记录系统数量快照和差异值，需另行发起库存调整才会真正改变库存 |
| **下架（Removal）** | `sorted_stock` ↑，`onhand/damage/return` 同步更新 | GoodsLocation 减少 → 触发重算；货物进入 sorted_stock 等待重新上架或处置 |
| **库存快照（Snapshot）** | 无 | 只读，将当前 GoodsLocation.quantity 按时间切片存档，用于历史查询 |

## 七、关键设计说明

以下设计在阅读代码时容易产生疑问，特此说明。

**1. `damage_stock` 和 `return_stock` 计入可用量**

`available_stock` 包含这两个字段，是有意设计。本系统中残次品和退货品均可正常出库，因此纳入可出库量计算。

**2. `delivered_stock` 不计入 `total_stock`**

货物一旦交给承运商，即视为离开仓库，不再计入仓库总库存。`delivered_stock` 只用于追踪"在途但尚未被客户签收"的状态，与仓库库存计算无关。

**3. `received_stock` 使用计划量而非实物量**

包裹签收时（`receive_asn`）并不清点实物，直接按 ASN 计划量增加 `received_stock`。实物数量以分拣结果（`actual_quantity`）为准，最终体现在 `sorted_stock` 中。因此 `received_stock` 是一个计划量占位符，在分拣完成后会用相同的计划量抵消归零，计划量与实物量的差异完全由 `actual_quantity` 吸收。

**4. `locked_stock` 当前未启用**

`lock_inventory` / `unlock_inventory` 方法已就绪，预留给将来的手动冻结功能（例如质检扣押、纠纷冻结等场景）。目前 `locked_stock` 始终为 0，`available_stock_for_sale` 实际等于 `onhand_stock - dn_stock`。

**5. `dn_stock` 按实际拣货量扣减**

拣货完成时（`dn_picked`），`dn_stock` 按**实际拣货量**（`picked_quantity`）扣减，而非原始预扣量。缺货拣不满时，未拣部分继续保留在 `dn_stock` 中，直到 DN 关闭后随聚合计算自动清除。

**6. `asn_stock` 和 `dn_stock` 的更新机制**

这两个字段不是增量 +/-，而是在相关单据创建/关闭时**重新从订单表聚合计算**（分别查询 pending 状态的 ASN/DN 明细求和）。因此它们始终与订单表保持一致，不存在累积误差。
